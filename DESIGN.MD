# Clutchly Landing Page Design System

> This document defines the complete design language, visual identity, layout principles, and UI rules for the Clutchly landing page. Every section should follow these guidelines to maintain consistency.

---

# About Clutchly

Clutchly is an AI-powered interview preparation platform that conducts realistic voice interviews, analyzes performance, remembers previous interviews, and continuously adapts to help users become better interviewers.

The product should feel:

- Premium
- Modern
- Intelligent
- Friendly
- Trustworthy
- Minimal
- Developer-focused

The website should communicate that Clutchly is **not another AI chatbot**, but a **personal AI interview coach that grows with you.**

---

# Overall Design Direction

The website should feel inspired by:

- Linear
- Perplexity
- Stripe
- Vercel
- Arc Browser
- Raycast
- Notion (minimalism)

Avoid flashy startup gradients.

Avoid clutter.

Everything should feel polished, calm, spacious and intentional.

# Buttons

Primary

- Purple background
- White text
- Rounded
- Medium shadow

Secondary

- White
- Border
- Dark text

Never use more than two CTA buttons together.

---

# Cards

Cards should always have:

- White background
- Rounded corners
- Soft shadows
- Large padding
- Minimal borders

Avoid:

- Dark cards
- Glassmorphism everywhere
- Heavy gradients

---

✓ Layered mockups

✓ Floating UI

✕ Bright gradients

✕ Busy illustrations

✕ Abstract blobs everywhere

# Copywriting Style

Short.

Clear.

Benefit-driven.

Avoid technical jargon.

Explain AI simply.

Good:

"The AI remembers every interview."

Bad:

"Persistent semantic memory graph powered by vector embeddings."

---

# Component Rules

Every component should be self-contained.

Never generate:

- Navbar
- Footer
- Previous sections
- Next sections

Unless explicitly requested.

Generate only the requested component.

---

# Responsive Design

Desktop First

Tablet

Mobile

Every section must scale beautifully.

Cards should stack naturally.

Typography should scale proportionally.

---

# Accessibility

Maintain strong contrast.

Large touch targets.

Readable typography.

Visible focus states.

Keyboard-friendly accordions.

---

# Overall Feeling

If someone visits Clutchly, they should immediately think:

"This feels like a premium AI product."

Not

"This looks like another AI SaaS template."

The landing page should balance simplicity with rich product visuals, making the AI interview experience feel tangible, trustworthy, and aspirational.
